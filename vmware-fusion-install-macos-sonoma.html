    How to Create a macOS Sonoma Virtual Machine in VMware Fusion

    Posted on 2024-07-07 Edited on 2024-07-18 In Virtual Machine Views: Word count in article: 1k Reading time ≈ 4 mins.
    I got backstabbed by Apple! My 2017 MBP can't upgrade to Sonoma, and since XCode is tied to macOS versions, no system upgrade means no XCode upgrade. The older XCode versions don't support the latest iOS, so I can't deploy apps to my iPhone, turning my old Mac into a pretty paperweight. I agonized over installing a Hackintosh, setting up KVM passthrough, or using a virtual machine for ages. In the end, I decided to install the latest XCode on a macOS Sonoma virtual machine. It's not ideal, but hey, it works!🐶🐶

    Please note: The software used in this post is VMware Fusion 13.5.2, the virtual machine system version is macOS Sonoma 14.5, and the host machine is a 2017 MacBook Pro running macOS Ventura 13.6.7. Please be aware of the timing.

    This blog post is translated from my Chinese blog post, the content in the images used in the post may still be in Chinese, sorry for the inconvenience.

    Activate VMware Fusion

    In May, VMware announced that it’s free for personal use, so when the activation page pops up during installation, you don’t need to enter an activation code. Just select the Personal Use option to activate it.

    Download macOS Installation Files

    Before starting the virtual machine installation, please obtain the macOS installation files. For detailed instructions, refer to my previous blog post: How to Download and Create a macOS Sonoma Installation ISO Image.

    Configure the Virtual Machine

    VMware Fusion natively supports installing macOS virtual machines, so unlike Workstation, you don’t need to use an unlocker. Just follow these steps, and leave any unspecified settings at their default values:

    1. On the Select the Installation Method page, select Create a custom virtual machine.
    2. On the Choose Operating System page, choose Apple OS X -> macOS 14.
    3. On the Choose a Virtual Disk page, choose Create a new virtual disk.
    4. Complete the virtual machine configuration.
    5. Do not start the virtual machine yet. If it has already started, shut it down completely (do not suspend it).
    6. Click the Settings button in the top menu bar to configure the VM. If the VM is running, the settings page will be grayed out and unchangeable.
    7. Adjust the CPU count, memory, and virtual hard disk size as needed.
    8. In CD/DVD(SATA), select the macOS installation image obtained earlier, and check the Connect CD/DVD Drive option.

    Install macOS VM

    1. Close the configuration page and start the virtual machine.
    2. Select Disk Utility.
    3. Click the top dropdown menu and choose Show All Devices.
    4. Select VMware Virtual SATA Hard Drive Media, rename it, and then erase it, keeping the default format and scheme.
    5. Follow the standard installation steps to install macOS Sonoma.

    Install VMware Tools

    1. After completing the installation and entering the desktop, right-click the installation image at the top right corner of the desktop and eject it.
    2. On the host machine, open the Virtual Machine dropdown menu from the top menu bar and select Install VMware Tools.
    3. Double-click Install VMware Tools to start the installation.
    4. During the installation, several permission request pop-ups will appear, including one that requires you to go into the settings. Allow all of them.
    5. Restart the virtual machine.

    Connect to the Network

    If there is no network connection, click the Settings button in the top menu bar, then select Network Adapter. Choose the Bridged Networking option under Network Connection, select Autodetect, and check the Connect network adapter box.

    About 3D Acceleration

    By default, macOS virtual machines cannot directly utilize the GPU, so all graphics are laboriously emulated by the CPU, resulting in noticeable lag. Additionally, Sonoma’s default desktop background is a video, which won’t play, leaving a blank white background. Fortunately, since our host machine is a real Mac, we can forcibly enable GPU 3D acceleration by editing the .vmx file. However, please note that this method is not perfect and may cause crashes. For those interested, you can check out my next blog post for details: Enable 3D Acceleration for macOS Virtual Machines in VMware Fusion.
